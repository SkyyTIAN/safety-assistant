ASIL

Automotive Safety Integrity Level

安全概念

汽车安全完整性等级，是ISO 26262标准中定义的功能安全风险分类等级。分为4级：ASIL A（最低）到ASIL D（最高），其中ASIL D对安全要求最严格。

ASIL A ASIL B ASIL C ASIL D

FIT

Failure In Time

芯片术语

失效率单位，表示每十亿小时（10⁹小时）的失效次数。1 FIT = 1次失效/10⁹小时。例如：10 FIT表示每十亿小时平均发生10次失效。

HARA

Hazard Analysis and Risk Assessment

分析方法

危害分析和风险评估，是功能安全开发的第一步。通过识别潜在危害、评估风险等级、确定安全目标，从而推导出安全需求。

FMEA

Failure Mode and Effects Analysis

分析方法

失效模式与影响分析，是一种系统性的分析方法，用于识别潜在的失效模式、评估其影响，并制定改进措施。分为DFMEA（设计）和FMEA（过程）。

DC

Diagnostic Coverage

安全概念

诊断覆盖率，表示安全机制能够检测到的失效占总失效的比例。通常分为：低(60-90%)、中(90-97%)、高(>97%)三个等级。DC是计算SPFM和LFM的关键参数。

SPFM

Single-Point Fault Metric

安全概念

单点故障度量，表示单点故障被安全机制覆盖的比例。计算公式：SPFM = 1 - (λ_SPFM / λ_total)。ASIL B要求≥60%，ASIL C≥70%，ASIL D≥90%。

LFM

Latent Fault Metric

安全概念

潜在故障度量，表示潜伏故障被安全机制覆盖的比例。潜伏故障是指无法被检测也不会立即导致故障的失效。ASIL B/C要求≥60%，ASIL D≥80%。

SBC

System Basic Chip / System Basis Chip

芯片术语

系统基础芯片，是汽车电子中常用的电源管理和监控芯片。SBC通常集成供电、 看门狗、安全状态控制等功能，是功能安全系统的重要组成部分。

MCU

Microcontroller Unit

芯片术语

微控制器单元，是汽车电子控制系统的核心。功能安全系统中使用的MCU通常需要具备硬件安全特性，如锁步核、内存ECC、看门狗等。

LBIST

Logic Built-In Self-Test

芯片术语

逻辑内建自检，是一种CPU逻辑自检机制。通过内部测试向量检测CPU逻辑单元的失效，提供高诊断覆盖率（通常>90%）。

MBIST

Memory Built-In Self-Test

芯片术语

内存内建自检，用于检测RAM/Flash中的物理失效。常见算法包括March测试、ATPG等，可提供>95%的诊断覆盖率。

FTA

Fault Tree Analysis

分析方法

故障树分析，是一种自顶向下的分析方法。从顶事件（不希望发生的事件）出发，通过逻辑门（与、或）分析导致顶事件的各种原因。
